# Chapter Specification Template

## Target Audience
- Computer Science students with Python programming experience
- New to robotics concepts and applications
- Prefer hands-on learning with visual aids
- Need clear explanations without assuming prior robotics knowledge

## Tone & Style
- Clear, accessible language focused on teaching first
- Visual and diagram-heavy explanations
- Practical examples over theoretical concepts
- Conversational but professional tone
- Step-by-step explanations with reasoning

## Required Chapter Sections

### 1. Concept
- Introduce the core idea in simple, relatable terms
- Explain why this concept matters in robotics
- Connect to prior knowledge (Python programming, general CS concepts)
- Use analogies to familiar CS concepts when possible

### 2. ASCII Diagram
- Include clear, well-labeled ASCII diagrams showing:
  - System architecture or component relationships
  - Data flow or process flow
  - Key mechanisms or operations
- Ensure diagrams are scalable and readable in text format
- Add brief annotations explaining each diagram element

### 3. Real-world Analogy
- Provide a relatable, everyday-life analogy that mirrors the concept
- Explain how the analogy maps to the technical concept
- Use examples from common experiences (transportation, communication, etc.)

### 4. Pseudo-code (ROS 2 / Python style)
- Write code examples in Python-style pseudo-code
- Follow ROS 2 conventions and patterns
- Include comments explaining the "why" behind each step
- Show both simple and slightly more complex examples
- Highlight important implementation details

### 5. Summary
- Recap the key points covered in the chapter
- Connect the concept to the broader robotics context
- Preview how this concept connects to upcoming topics

### 6. Exercises (3-5)
- Include 3-5 hands-on exercises of varying difficulty:
  - 1-2 basic understanding exercises
  - 1-2 application exercises
  - 1 challenge exercise for advanced learners
- Provide clear instructions and expected outcomes
- Include hints or guidance where appropriate

## Content Guidelines to Follow

### DO:
- Use simple, direct language
- Include visual elements (diagrams, tables, examples)
- Provide step-by-step explanations
- Connect concepts to Python programming where possible
- Include practical, hands-on examples
- Focus on understanding over memorization

### AVOID:
- Research paper language or academic jargon
- Heavy mathematical explanations without intuition
- Unexplained technical terms
- Assumptions about prior robotics knowledge
- Dense theoretical content without practical context
- Complex notation without proper introduction
